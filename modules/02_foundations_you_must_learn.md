# Module 2: Foundations You Must Learn

## Core Technical Stack

These are the topics almost every serious AI transition needs.

### 1. Python

You should be comfortable with:

1. functions, classes, modules, packages
2. file I/O and JSON
3. virtual environments and dependency management
4. debugging and reading stack traces
5. notebooks and scripts

### 2. Data Work

You should understand:

1. tabular data and CSV/Parquet basics
2. missing values and bad data
3. joins, filtering, grouping, aggregation
4. train/validation/test splits
5. leakage and data quality problems

### 3. Math And Stats At The Right Depth

You do not need to become a theoretician first, but you do need practical fluency in:

1. probability intuition
2. averages, variance, and distributions
3. precision, recall, F1, and confusion matrices
4. gradient descent intuition
5. vectors and matrices
6. cosine similarity and embeddings intuition

### 4. Software Engineering Basics

If you are not already strong here, learn:

1. Git and pull request workflows
2. testing basics
3. APIs and HTTP
4. environment configuration
5. packaging and deployment basics

## What Depth Is Actually Needed?

Use this rule of thumb:

- You need enough math to understand model behavior and debugging.
- You need enough Python to build and modify AI applications yourself.
- You need enough statistics to evaluate whether something is actually working.

You do not need to master every proof before building useful systems.

## Research-Backed Learning Direction

Public learning programs consistently emphasize these foundations:

1. Google ML Crash Course highlights regression, classification, data handling, overfitting, embeddings, LLM basics, and production ML systems.
2. fast.ai emphasizes that people with coding skill can learn deep learning effectively through practical projects and staged theory.
3. DeepLearning.AI's introductory material stresses understanding what AI can and cannot do, along with project workflow.

## Minimum Evidence That You Learned The Foundations

You should be able to:

1. read and modify Python project code
2. load a dataset and clean obvious issues
3. train a simple model or call an LLM app pipeline with understanding
4. explain at least three evaluation metrics and when they matter
5. debug a small project failure without panic
