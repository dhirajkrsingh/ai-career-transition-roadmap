# Module 10: Interview Question Banks By Target Role

## Why This Module Exists

Many people can learn enough to build a project, but they are not prepared to defend their decisions in an interview.

The goal here is not memorization. The goal is to train the ability to explain trade-offs clearly.

## AI Engineer Interview Questions

1. Walk me through an AI application you built end to end.
2. Why did you choose that model or prompting strategy?
3. How did you ground the system to reduce hallucinations?
4. How did you evaluate whether it was actually useful?
5. What were the major failure modes?
6. How would you improve latency, cost, or accuracy?
7. When would you choose a workflow or agent design over a single prompt?

## ML Engineer Interview Questions

1. How did you choose evaluation metrics for your model?
2. How did you handle overfitting or data leakage risk?
3. What baseline did you compare against?
4. How did you move from notebook experimentation to a repeatable workflow?
5. What would you monitor after deployment?
6. How would you debug degraded model quality in production?

## Data Scientist Interview Questions

1. How did you frame the business or product problem?
2. Why was the chosen metric the right one?
3. How did you validate that your model was useful, not just statistically interesting?
4. What assumptions did you make about the data?
5. How did you communicate trade-offs to non-technical stakeholders?

## MLOps / Platform Interview Questions

1. How would you productionize an ML or LLM system safely?
2. What would you log and monitor?
3. How would you version prompts, models, and evaluation cases?
4. How would you detect regressions after a prompt or model upgrade?
5. How would you make the workflow reproducible for another engineer?

## AI Product Manager Interview Questions

1. How do you decide whether AI is the right solution to a product problem?
2. What would success metrics look like for an AI feature?
3. How would you handle model unreliability in a user-facing product?
4. How would you scope an MVP for an AI feature?
5. How would you coordinate engineering, data, design, and policy concerns?

## Reliability / Evaluation Interview Questions

1. How do you test an LLM system for groundedness?
2. How do you distinguish hallucination from acceptable uncertainty?
3. What cases would you include in a regression suite?
4. How would you evaluate a prompt improvement claim?
5. What failure should block release?

## How To Practice These Questions

For each project in your portfolio, prepare:

1. one architecture explanation
2. one metric explanation
3. one failure story
4. one trade-off story
5. one next-step improvement story

If you cannot answer those five clearly, the project is not yet interview-ready.
