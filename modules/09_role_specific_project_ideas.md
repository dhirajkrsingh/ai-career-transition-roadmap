# Module 9: Role-Specific Project Ideas

## Why This Module Exists

Most career switchers know they need projects, but they choose projects that are either too generic, too large, or not aligned with the role they want.

This module gives project directions that map to actual AI role signals.

## For AI Engineer

### Project 1: Domain Knowledge Assistant

Build:

- a retrieval-based assistant over a real domain knowledge base

Show:

1. ingestion pipeline
2. retrieval strategy
3. prompt constraints
4. evaluation cases
5. basic deployment

### Project 2: Workflow Automation Agent

Build:

- an agent or multi-step workflow that turns unstructured input into action

Examples:

- support triage assistant
- internal policy lookup and summarization workflow
- code change analysis assistant

### Project 3: AI Product Prototype

Build:

- an end-to-end app with frontend, backend, and AI logic

Show:

1. user journey
2. model or prompt decisions
3. cost and latency trade-offs

## For ML Engineer

### Project 1: Classical ML Prediction Project

Build:

- a prediction or classification workflow with clean train/validation/test handling

Show:

1. feature preparation
2. metric choice
3. baseline vs improved model
4. error analysis

### Project 2: Model Serving Pipeline

Build:

- a trained model exposed behind an API or service

Show:

1. inference interface
2. reproducibility
3. deployment or serving logic

### Project 3: Evaluation Dashboard Or Monitoring Concept

Build:

- a small tool that tracks model or system quality over time

## For Data Scientist

### Project 1: Business Problem Modeling Project

Build:

- a project tied to forecasting, classification, ranking, or risk estimation

Show:

1. problem framing
2. metric choice
3. assumptions and caveats
4. business interpretation

### Project 2: LLM-Augmented Analytics Assistant

Build:

- a tool that helps summarize, query, or analyze internal documents or structured data

Show:

1. reasoning limits
2. evaluation plan
3. failure cases

## For MLOps / AI Platform Engineer

### Project 1: Reproducible Training And Deployment Workflow

Build:

- a small pipeline for training, packaging, and serving a model or AI app

Show:

1. environment management
2. deployment path
3. monitoring hooks

### Project 2: Prompt / Model Eval Harness

Build:

- an evaluation runner for prompt versions or model outputs

Show:

1. test cases
2. pass/fail logic
3. regression tracking

## For AI Product Manager

### Project 1: AI Feature Case Study

Create:

- a structured case study for one AI product opportunity

Show:

1. user problem
2. why AI is justified
3. evaluation criteria
4. rollout risks

### Project 2: Low-Code Or Code-Light Prototype

Build:

- a small working prototype that proves you understand interaction and constraints

## For AI Evaluation / Reliability Roles

### Project 1: Hallucination Benchmark Pack

Build:

- a test suite for groundedness, uncertainty, and format compliance

### Project 2: Prompt Regression Pipeline

Build:

- a small framework to compare prompt versions and report regressions

## Selection Rule

Pick projects that score well on these four questions:

1. Does it match the target role?
2. Does it show real decision-making?
3. Can it be completed in reasonable time?
4. Can you explain its evaluation and failure modes?
